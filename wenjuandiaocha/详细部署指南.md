# 详细部署指南：云护童心网站

## 适用人群

本指南专为**完全不了解技术**的用户设计，包含了从安装必要工具到最终部署网站的**每一个详细步骤**。请按照指南一步一步操作，确保每一步都成功完成后再进行下一步。

## 准备工作

### 步骤 1：下载并安装 Git

Git 是一个版本控制工具，用于将代码上传到 GitHub。

1. **下载 Git 安装包**：
   - 访问 [Git 官网](https://git-scm.com/download/win)
   - 下载适合您系统的版本（通常是 "64-bit Git for Windows Setup"）

2. **安装 Git**：
   - 双击下载的安装包开始安装
   - 点击 "Next" 直到出现 "Select Components" 页面
   - 保持默认选项，继续点击 "Next"
   - 在 "Adjusting your PATH environment" 页面，选择 "Git from the command line and also from 3rd-party software"
   - 继续点击 "Next" 直到安装完成
   - 点击 "Finish" 完成安装

3. **验证 Git 安装**：
   - 按下 `Win + R` 键打开运行对话框
   - 输入 `cmd` 并按 Enter 键打开命令提示符
   - 在命令提示符中输入 `git --version` 并按 Enter
   - 如果看到 Git 版本信息（如 "git version 2.43.0.windows.1"），则安装成功

### 步骤 2：创建 GitHub 账号

GitHub 是一个代码托管平台，用于存储和管理网站代码。

1. **访问 GitHub 官网**：
   - 打开浏览器，访问 [GitHub官网](https://github.com)

2. **注册账号**：
   - 点击右上角的 "Sign up"
   - 输入您的邮箱地址（建议使用常用邮箱）
   - 点击 "Continue"
   - 创建一个密码（至少 15 个字符，或包含数字和字母）
   - 点击 "Continue"
   - 输入一个用户名（如 "yunhutongxin-user"）
   - 点击 "Continue"
   - 选择 "Skip personalization"（跳过个性化设置）
   - 验证您不是机器人（按照提示完成验证）
   - 登录您的邮箱，点击 GitHub 发送的验证邮件中的链接

3. **登录 GitHub**：
   - 回到 GitHub 官网，点击 "Sign in"
   - 输入您的邮箱和密码，点击 "Sign in"

### 步骤 3：创建 GitHub 仓库

仓库是存储网站代码的地方。

1. **创建新仓库**：
   - 登录 GitHub 后，点击右上角的 "+" 按钮
   - 选择 "New repository"

2. **填写仓库信息**：
   - **Repository name**：输入 `yunhutongxin-website`
   - **Description**：输入 `云护童心 - 儿童心理健康成长调查系统`
   - **Visibility**：选择 "Public"（公开）
   - 勾选 "Add a README file"
   - 点击 "Create repository"

3. **复制仓库地址**：
   - 进入新创建的仓库页面
   - 点击绿色的 "Code" 按钮
   - 复制显示的 HTTPS 地址（如 "https://github.com/your-username/yunhutongxin-website.git"）

## 上传网站文件

### 步骤 4：克隆仓库到本地

1. **打开 Git Bash**：
   - 按下 `Win + R` 键打开运行对话框
   - 输入 `git bash` 并按 Enter 键

2. **克隆仓库**：
   - 在 Git Bash 中输入以下命令（将 `your-username` 替换为您的 GitHub 用户名）：
     ```bash
     git clone https://github.com/your-username/yunhutongxin-website.git
     ```
   - 按 Enter 键执行命令
   - 当提示输入 GitHub 用户名和密码时：
     - 输入您的 GitHub 用户名
     - 输入您的 GitHub 密码（注意：输入密码时不会显示任何字符，这是正常的）
     - 按 Enter 键

3. **进入仓库目录**：
   - 克隆完成后，输入以下命令：
     ```bash
     cd yunhutongxin-website
     ```
   - 按 Enter 键执行命令

### 步骤 5：复制网站文件

1. **找到网站文件**：
   - 打开文件资源管理器，导航到 `C:\Users\zszs7\Documents\trae_projects\wenjuandiaocha` 文件夹
   - 您会看到以下文件：
     - `index.html`
     - `script.js`
     - `styles.css`
     - `README.md`
     - `详细部署指南.md`
     - `.gitignore`

2. **复制文件到仓库**：
   - 选择所有文件（按 `Ctrl + A`）
   - 右键点击选中的文件，选择 "复制"
   - 导航到 `C:\Users\zszs7\Documents\trae_projects\wenjuandiaocha\yunhutongxin-website` 文件夹
   - 右键点击文件夹空白处，选择 "粘贴"
   - 如果提示是否替换文件，选择 "替换所有文件"

### 步骤 6：提交并推送文件

1. **添加文件到 Git**：
   - 回到 Git Bash 窗口
   - 输入以下命令：
     ```bash
     git add .
     ```
   - 按 Enter 键执行命令

2. **提交文件**：
   - 输入以下命令：
     ```bash
     git commit -m "上传网站文件"
     ```
   - 按 Enter 键执行命令

3. **推送到 GitHub**：
   - 输入以下命令：
     ```bash
     git push origin main
     ```
   - 按 Enter 键执行命令
   - 当提示输入 GitHub 用户名和密码时，再次输入并按 Enter

4. **验证推送**：
   - 打开浏览器，访问您的 GitHub 仓库页面
   - 刷新页面，您应该能看到所有上传的文件

## 部署到 Vercel

### 步骤 7：创建 Vercel 账号

Vercel 是一个静态网站托管平台，用于将网站部署到互联网上。

1. **访问 Vercel 官网**：
   - 打开浏览器，访问 [Vercel官网](https://vercel.com)

2. **使用 GitHub 账号登录**：
   - 点击 "Sign Up"
   - 选择 "Continue with GitHub"
   - 输入您的 GitHub 用户名和密码（如果需要）
   - 点击 "Authorize Vercel"

### 步骤 8：部署项目

1. **导入 GitHub 仓库**：
   - 登录 Vercel 后，点击 "New Project"
   - 在 "Import Git Repository" 部分，搜索 `yunhutongxin-website`
   - 点击您的仓库名称

2. **配置部署选项**：
   - 项目名称：保持默认（`yunhutongxin-website`）
   - Framework Preset：选择 "Other"
   - Root Directory：保持默认（`./`）
   - Build Command：留空
   - Output Directory：留空
   - 点击 "Deploy"

3. **等待部署完成**：
   - Vercel 会自动构建和部署您的网站
   - 部署过程中会显示 "Building" 和 "Deploying" 状态
   - 当看到 "Congratulations! Your deployment is live." 时，部署完成

### 步骤 9：访问部署后的网站

1. **获取网站链接**：
   - 部署完成后，Vercel 会显示一个 "Visit" 按钮
   - 点击该按钮，或复制显示的链接（如 `https://yunhutongxin-website.vercel.app`）

2. **访问网站**：
   - 在浏览器中打开复制的链接
   - 您应该能看到 "云护童心" 网站的首页

3. **验证网站功能**：
   - 尝试点击 "学生" → "开始使用"
   - 填写测评问卷并提交
   - 查看测评结果和资源包
   - 尝试其他功能，确保网站正常运行

## 常见问题解决

### 问题 1：Git 安装失败

**症状**：安装 Git 时出现错误

**解决方法**：
- 重新下载 Git 安装包，确保下载的是完整的安装包
- 关闭所有其他程序，然后重新安装
- 如果仍然失败，尝试下载旧版本的 Git

### 问题 2：无法克隆 GitHub 仓库

**症状**：执行 `git clone` 命令时出现错误

**解决方法**：
- 检查您的网络连接是否正常
- 确保输入的仓库地址正确
- 确保您的 GitHub 用户名和密码正确
- 如果使用的是 GitHub 令牌（Token），确保令牌有正确的权限

### 问题 3：推送文件到 GitHub 失败

**症状**：执行 `git push` 命令时出现错误

**解决方法**：
- 检查您的网络连接是否正常
- 确保您有仓库的推送权限
- 尝试重新输入 GitHub 用户名和密码

### 问题 4：Vercel 部署失败

**症状**：Vercel 部署过程中出现错误

**解决方法**：
- 检查您的 GitHub 仓库中是否包含所有必要的文件（index.html, script.js, styles.css）
- 查看 Vercel 部署日志，了解具体的错误信息
- 尝试重新部署项目

### 问题 5：网站无法访问

**症状**：部署后无法访问网站，或网站显示错误

**解决方法**：
- 检查 Vercel 部署状态，确保部署成功
- 尝试清除浏览器缓存后重新访问
- 检查网站文件是否完整，特别是 index.html 文件
